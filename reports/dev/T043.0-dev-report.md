# T043.0 开发报告 — 修复 run-project-next 的 Claude Code 超时异常处理

## 任务信息

- 任务编号：T043.0
- 角色：Developer
- 状态：完成

## 问题背景

执行 G005 时，Claude Code 子进程超过 600 秒未结束，触发 `subprocess.TimeoutExpired`，导致 runner.py 直接崩溃。
没有保存清晰失败报告，没有写入 latest-output.md，没有友好提示用户如何恢复，子项目任务可能停留在 in_progress。

## 修改文件

| 文件 | 改动说明 |
|------|----------|
| `tools/claude_code_runner.py` | 捕获 `TimeoutExpired`，返回 `returncode=124`，增加 `timed_out` / `timeout_seconds` 字段 |
| `tools/project_runner.py` | 失败路径传播 `timed_out` / `timeout_seconds` 到返回结果 |
| `runner.py` | `save_latest_output` 增加超时信息写入；`_handle_run_project_next` 区分超时和普通失败 |
| `docs/tasks.md` | 追加 T043.0 |
| `reports/dev/T043.0-dev-report.md` | 新建 — 开发报告 |

## 完成内容

- 在 `run_claude_code()` 中捕获 `subprocess.TimeoutExpired`
- 超时时返回 `returncode=124`、`timed_out=True`、`timeout_seconds=600`
- 定义常量 `CLAUDE_CODE_TIMEOUT_SECONDS = 600`
- `save_latest_output()` 写入 `## Timed Out` 和 `## Timeout Seconds` 段落
- `_handle_run_project_next()` 超时时输出具体原因和恢复建议
- `run_project_next()` 失败路径传播 `timed_out` 字段
- 正常执行结果增加 `timed_out=False`、`timeout_seconds=None`（向后兼容）
- 任务状态保持 `in_progress`，不自动重试
