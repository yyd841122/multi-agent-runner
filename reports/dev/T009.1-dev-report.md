# T009.1 开发报告 — 修复 run-next 成功误判问题

## 问题原因

`run-next` 和 `auto-complete-success` 只根据 Claude Code 的 returncode == 0 判断任务完成。但 Claude Code 退出码为 0 不代表它真正完成了任务（可能只输出了建议代码而没有实际修改文件），导致任务被错误标记为 done。

## 修改文件列表

| 文件 | 改动说明 |
|------|----------|
| `docs/tasks.md` | 追加 T009.1 任务定义 |
| `tools/report_manager.py` | 新增 `has_completion_evidence(task, reports_dir)` |
| `runner.py` | `run_next()` 和 `auto_complete_success()` 增加 `has_completion_evidence` 检查 |
| `reports/dev/T009.1-dev-report.md` | 新建开发报告 |

## 修复内容

### tools/report_manager.py

新增 `has_completion_evidence(task, reports_dir)`：检查 `reports/dev/<任务编号>-dev-report.md` 是否存在。

### runner.py — run_next()

原逻辑：`if analysis["success"]` → 自动 done
新逻辑：`if analysis["success"] and has_completion_evidence(task)` → 自动 done
- 成功但无证据 → 保持 in_progress，输出缺少文件提示
- 成功且有证据 → done

### runner.py — auto_complete_success()

同步修复：成功 + 有证据 → done；成功但无证据 → 不 done

## 验收标准自查

- [x] run-next 不再只根据 returncode=0 判断任务完成
- [x] 自动 done 前必须检查 reports/dev/<任务编号>-dev-report.md 是否存在
- [x] 如果执行成功但缺少开发报告，不自动标记 done
- [x] 缺少完成证据时给出明确提示
- [x] T010 不应被错误标记为 done

## 注意事项
