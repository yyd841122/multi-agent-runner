# T009.3 开发报告 — 修复 Claude Code 自动写文件权限

## 问题原因

`claude_code_runner.py` 调用 Claude Code 时使用 `claude --print` 模式，默认不开启文件编辑权限。Claude Code 在非交互模式下无法自动修改文件，只能输出建议代码，导致任务实际上没有完成却被误判为成功。

## 修改文件列表

| 文件 | 改动说明 |
|------|----------|
| `tools/claude_code_runner.py` | subprocess 调用增加 `--permission-mode acceptEdits` |
| `docs/tasks.md` | 追加 T009.3 任务定义 |
| `reports/dev/T009.3-dev-report.md` | 新建开发报告 |

## 修复内容

```python
# 修复前
[command, "--print", prompt]

# 修复后
[command, "--permission-mode", "acceptEdits", "--print", prompt]
```

`--permission-mode acceptEdits` 让 Claude Code 在非交互模式下自动接受文件编辑，不再需要手动授权。

## 验收标准自查

- [x] run_claude_code() 调用 Claude Code 时包含 --permission-mode acceptEdits
- [x] 保持原有 --print 调用方式
- [x] 保持原有 stdout/stderr/returncode 返回结构
- [x] 保持 encoding="utf-8", errors="replace"
- [x] 不修改 T010 状态
- [x] 不在 Claude Code 会话内执行 retry-current

## 注意事项

修复后需在普通 PowerShell 中执行 `python runner.py retry-current` 验证，不要在 Claude Code 会话内运行以避免嵌套调用。
