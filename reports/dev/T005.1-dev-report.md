# T005.1 开发报告 — 修复 Windows 下 Claude Code 输出编码问题

## 问题原因

Windows 下 Python `subprocess.run(..., text=True)` 默认使用系统编码（gbk）解码 stdout/stderr。Claude Code 输出包含 UTF-8 中文字符，gbk 无法解码，导致 `UnicodeDecodeError`。

## 修改文件

| 文件 | 改动说明 |
|------|----------|
| `tools/claude_code_runner.py` | subprocess.run 增加 `encoding="utf-8"` 和 `errors="replace"` |

## 修复内容

```python
# 修复前
subprocess.run(..., text=True)

# 修复后
subprocess.run(..., text=True, encoding="utf-8", errors="replace")
```

- `encoding="utf-8"` — 明确使用 UTF-8 解码，与 Claude Code 输出编码一致
- `errors="replace"` — 遇到无法解码的字符用占位符替代，避免程序崩溃

## 验证结果

运行 `python runner.py run-current`，不再出现 UnicodeDecodeError，退出码正常显示。
