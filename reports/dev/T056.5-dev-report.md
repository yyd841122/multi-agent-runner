## Implementation

### ReworkResumeResult

新增数据结构，包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| task_id | str | 返工任务编号 |
| rework_round | int | 返工轮次 |
| candidate_status | str | validated / missing / failed |
| execution_mode | str | resume_stub |
| execution_allowed | bool | 是否允许执行 |
| real_execution_performed | bool | 是否已真实执行 |
| resume_requested | bool | 用户是否传入 --resume |
| resume_allowed | bool | 是否允许恢复主流程 |
| resume_target | str | NEXT_PENDING / BLOCKED |
| resume_reason | str | 原因说明 |
| loop_status | str | resume_stub_ready / resume_blocked |
| safety_status | str | pass / fail |
| next_action | str | 建议下一步 |
| message | str | 详细消息 |

### prepare_full_loop_resume()

核心函数。复用 `execute_confirmed_rework()` 的全部安全检查逻辑：

1. 调用 `execute_confirmed_rework()` 执行完整校验
2. 如果 `execution_allowed=True` 且 `safety_status=pass`：
   - `resume_allowed=True`, `resume_target=NEXT_PENDING`, `loop_status=resume_stub_ready`
3. 如果任一检查未通过：
   - `resume_allowed=False`, `resume_target=BLOCKED`, `loop_status=resume_blocked`
   - 根据失败类型映射 resume_reason

### execute-rework --resume

在 runner.py 的 execute-rework 命令中新增 `--resume` 参数。

参数依赖关系：
- `--resume` 必须配合 `--real-execution` 使用
- `--resume` 必须配合 `--confirm` 使用
- 单独使用 `--resume` 返回错误提示

## Behavior

### 不带 --resume

保持 T056.2 原有行为不变：
- `--real-execution` → confirmed_rework_execution_stub
- 无 `--real-execution` → dry-run

### 带 --resume 但 safety gate fail

全部拒绝场景：

| 场景 | resume_reason |
|------|---------------|
| 缺少 --real-execution | 错误：--resume 必须配合 --real-execution 使用 |
| 缺少 --confirm | 错误：--resume 必须配合 --confirm 使用 |
| fuzzy confirm | confirmation rejected: 确认格式不匹配 |
| round < 1 | round invalid: 返工轮次不能小于 1 |
| round > 3 | round exceeded: 超过最大返工次数 3 |
| prompt 不存在 | safety check failed: rework prompt 不存在 |

### 带 --resume 且 confirmed stub pass

```
resume_allowed=True
resume_target=NEXT_PENDING
resume_reason=all safety checks passed
loop_status=resume_stub_ready
EXECUTION_MODE=resume_stub
candidate_status=validated
execution_allowed=True
real_execution_performed=False
CHECK_RESULT=pass
NEXT_ACTION=ready_for_next_pending_task
```

## Safety Rules

resume 永不执行的操作：

1. 调用 Claude Code
2. 修改业务代码（index.html / style.css / script.js）
3. 自动执行下一个 pending task
4. 自动修改任务状态
5. 自动进入 run-project-task-full
6. 无限循环调度
