# T043.1 开发报告 — 修复 run-project-next 返回码与完成证据冲突时的状态判断

## 任务信息

- 任务编号：T043.1
- 角色：Developer
- 状态：完成

## 问题背景

执行 G005 时，Claude Code 实际完成任务并生成 G005-dev-report.md，子项目任务状态已变为 done，但最后返回 API 429 错误导致 returncode=1。runner 输出"执行失败"，与实际完成状态不一致。

## 修改文件

| 文件 | 改动说明 |
|------|----------|
| `tools/project_runner.py` | `run_project_next()` 失败路径增加证据冲突检查：returncode!=0 但 evidence 存在且任务 done 时返回 `completed_with_model_error` |
| `runner.py` | `_handle_run_project_next()` 增加 `completed_with_model_error` 分支，输出"完成证据存在但模型返回错误"及人工确认建议 |
| `docs/tasks.md` | 追加 T043.1 并标记 done |
| `reports/dev/T043.1-dev-report.md` | 新建 — 开发报告 |

## 完成内容

- 在 `run_project_next()` 失败路径中增加二次检查：重新读取任务文件，判断任务是否已被 Claude Code 标记为 done
- 新增 `completed_with_model_error` 返回字段
- 新增 `model_returncode` 字段，记录实际模型返回码
- `_handle_run_project_next()` 增加冲突状态输出，明确提示人工确认
- 保持不自动重试、不修改小游戏业务代码

## 证据冲突判断逻辑

```
if returncode != 0:
    if dev_report_exists:
        re-read task status
        if task_status == "done":
            → completed_with_model_error（模型返回错误，但任务实际完成）
        else:
            → normal failure
    else:
        → normal failure
```
