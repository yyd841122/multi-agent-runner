# T056.2 Dev Report

## Task

实现 confirmed rework execution

## Scope

本轮只实现 confirmed execution stub，不真实调用 Claude Code，不修改业务代码。

## Changed Files

- tools/rework_manager.py（新增 ReworkConfirmedExecutionResult + execute_confirmed_rework）
- runner.py（新增 --real-execution 参数和 confirmed execution 分支）
- docs/tasks.md（追加 T056.2 记录）
- reports/dev/T056.2-dev-report.md（本文件）

## Behavior

### dry-run allow（无 --real-execution）

沿用 T056 已有的 `prepare_rework_execution` 逻辑：

- 输出 Status / Reason / Report
- `execution_allowed` 不在此路径直接显示
- `real_execution_performed=false`（隐式，不调用 Claude Code）

### confirmed execution stub（有 --real-execution）

新增 `execute_confirmed_rework` 逻辑：

- 输出结构化字段：`execution_allowed` / `real_execution_requested` / `real_execution_performed` / `EXECUTION_MODE` / `CHECK_RESULT` / `confirmation_status` / `round_status` / `Message` / `NEXT_ACTION`
- 全部检查通过时：`execution_allowed=True`, `real_execution_performed=False`, `EXECUTION_MODE=confirmed_rework_execution_stub`
- 未通过时：`execution_allowed=False`, `real_execution_performed=False`

### 区别

| 场景 | 走哪个函数 | execution_mode |
|------|-----------|----------------|
| 无 --real-execution | prepare_rework_execution | dry_run（T056 原有） |
| 有 --real-execution | execute_confirmed_rework | dry_run 或 confirmed_rework_execution_stub |

## Safety Rules

拒绝场景（全部返回 `execution_allowed=False`）：

1. 缺少 confirm → `confirmation_status=missing`
2. 模糊 confirm（"继续"等）→ `confirmation_status=rejected`
3. confirm 格式不匹配 → `confirmation_status=rejected`
4. confirm 与 task/round 不一致 → `confirmation_status=rejected`
5. round < 1 → `round_status=invalid`
6. round > 3 → `round_status=exceeded`
7. rework prompt 不存在 → `safety_status=fail`

## Verification

|---|------|------|------|------|

*注：G999 通过是因为 rework_prompt.md 文件存在（G007 生成的）。当前 MVP 只检查文件存在性，不校验 prompt 内容与 task_id 匹配。后续可增强。
