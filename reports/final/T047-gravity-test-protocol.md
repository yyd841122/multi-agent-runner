# T047 重力下落测试协议报告

## 1. 背景

第五阶段第一个任务是 G006 简单重力下落。

重力下落是游戏核心物理基础，后续碰撞检测、平台滚动、失败条件都依赖重力系统。

如果重力逻辑不正确，后续所有功能都会受影响。

因此，在 G006 实现前，需要先定义重力测试协议。

## 2. 当前问题

### 2.1 基础静态检查的局限

T030 的基础静态检查只能验证文件结构和关键元素是否存在，无法验证重力下落逻辑。

### 2.2 行为静态检查的局限

T036 的行为静态检查只能验证键盘左右移动逻辑，无法验证玩家是否随时间向下移动。

### 2.3 缺失的重力验证能力

当前 Tester 还不能验证：

- 玩家是否会随时间向下移动
- 是否存在重力常量
- 是否存在垂直速度变量
- 是否存在重力更新逻辑
- 重力更新是否在游戏循环中调用

## 3. 协议目标

定义 Tester Agent 如何检查 G006 简单重力下落逻辑：

- 定义 5 组共 15 个检查项
- 明确 PASS / FAIL / BLOCKED 判定规则
- 定义重力测试报告格式和完成证据路径
- 为 T050 实现 G006 完整闭环提供测试基础

## 4. 测试范围

### 4.1 测试目标

G006 只测试简单重力下落：

- 玩家随时间向下移动
- 存在 `GRAVITY` 重力值
- 存在 `velocityY` 垂直速度
- 存在重力更新逻辑
- 在 `gameLoop` 中调用重力更新
- 玩家 `y` 坐标随时间增加

### 4.2 不测试什么

- 平台碰撞
- 站在平台上
- 平台滚动
- 随机平台
- 游戏失败
- 游戏胜利
- 角色技能系统
- 微信小游戏 / 抖音小游戏适配

## 5. 检查项设计

共 5 组 15 个检查项：

### G 组：重力常量 / 状态检查（3 项）

| 编号 | 测试项 | 检查目标 | 必需 |
|------|--------|----------|------|
| G-01 | 重力常量存在 | `GRAVITY` 或 `gravity` | 是 |
| G-02 | 玩家垂直速度存在 | `velocityY` 或 `vy` 或 `playerState.velocityY` | 是 |
| G-03 | 玩家 y 坐标状态存在 | `playerState.y` 或 `playerY` | 是 |

### V 组：垂直速度更新检查（3 项）

| 编号 | 测试项 | 检查目标 | 必需 |
|------|--------|----------|------|
| V-01 | 重力影响速度 | `velocityY += GRAVITY` 或类似逻辑 | 是 |
| V-02 | 速度用于位置更新 | `playerState.y += velocityY` 或类似逻辑 | 是 |
| V-03 | 垂直速度有初始值 | `velocityY: 0` 或初始化赋值 | 是 |

### Y 组：垂直位置更新检查（3 项）

| 编号 | 测试项 | 检查目标 | 必需 |
|------|--------|----------|------|
| Y-01 | 玩家 top 更新存在 | `style.top` | 是 |
| Y-02 | 玩家 y 坐标会变化 | `playerState.y +=` 或 `playerY +=` | 是 |
| Y-03 | 下落方向正确 | y 值增加表示向下移动 | 是 |

### L 组：游戏循环集成检查（3 项）

| 编号 | 测试项 | 检查目标 | 必需 |
|------|--------|----------|------|
| L-01 | 游戏循环存在 | `gameLoop` 或 `requestAnimationFrame` | 是 |
| L-02 | 重力更新在循环中调用 | `applyGravity()` / `updateGravity()` / `handleGravity()` | 是 |
| L-03 | 位置更新在循环中调用 | `updatePlayerPosition()` | 是 |

### S 组：范围限制与非目标检查（3 项）

| 编号 | 测试项 | 检查目标 | 必需 |
|------|--------|----------|------|
| S-01 | 不包含平台碰撞逻辑 | 不应出现复杂 `collision` / `isOnPlatform` | 是 |
| S-02 | 不包含平台滚动逻辑 | 不应出现 `scrollPlatforms` 等逻辑 | 是 |
| S-03 | 不包含游戏失败条件 | 不应出现 `gameOver` / `fail` 判定 | 是 |

## 6. PASS / FAIL / BLOCKED 规则

| 条件 | 结果 |
|------|------|
| 全部必需项通过 | `PASS` |
| 任一必需项失败 | `FAIL` |
| `script.js` 不存在或无法读取 | `BLOCKED` |
| 范围越界实现明显存在（S 组 FAIL） | `FAIL` |

结果枚举：

- `PASS`：全部必需项通过
- `FAIL`：至少一个必需项失败
- `BLOCKED`：无法执行测试

## 7. 完成证据路径

重力行为检查报告保存路径：

```text
<project-root>/reports/test/<task-id>-gravity-test-report.md
```

例如：

```text
projects/down-100-floors-game/reports/test/G006-gravity-test-report.md
```

报告模板：`templates/agent-output/tester-gravity-check-template.md`
