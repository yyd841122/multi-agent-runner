## 1. 规划背景

`multi-agent-runner` 已完成两个阶段：

- **第一阶段**：最小自动化执行闭环（T001-T011）
- **第二阶段**：协议化 + 真实子项目验证（T013-T022.1）

- Planner Agent / Reviewer Agent / Main Agent 雏形
- down-100-floors-game 验证项目（G001 / G002 已完成）

## 2. 第三阶段核心定位

**第三阶段不是"做完整小游戏"，而是用小游戏继续验证和强化框架自动化能力。**

第三阶段聚焦于把第二阶段的 MVP 能力向"更通用、更真实、更可验证"推进，具体包括：

1. **通用化** — `run-game-next` → 通用 `run-project-next`
2. **真实化** — Reviewer 从 mock → 真实国内模型
3. **结构化** — Reviewer 输出可机器解析
4. **可验证** — 加入 Tester Agent 最小静态检查
5. **闭环化** — Main Agent 综合三方结果做决策

## 5. 与最终目标的对齐

最终目标：用户提出需求 → 系统自动拆解 → 自动执行 → 自动测试 → 自动审查 → 自动返工 → 用户只做验收。

第三阶段贡献：

| 自动化测试 | T029 / T030（Tester Agent） |
| 综合智能决策 | T031（三方综合决策） |

## 7. 预期里程碑

第三阶段完成后，`multi-agent-runner` 将具备：

1. 通用项目执行器（可服务任意子项目）
2. 真实模型驱动的 Reviewer Agent
3. 结构化审查结果（可被 Main Agent 消费）
4. Tester Agent 最小静态检查
5. Developer + Tester + Reviewer 三方综合决策
6. 一个通过完整自动化链路的游戏功能（G003）

这将使框架从"能自动写代码"进化到"能自动写代码、自动测试、自动审查、自动决策"。
