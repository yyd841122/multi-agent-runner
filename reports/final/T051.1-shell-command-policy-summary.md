# T051.1 Bash / PowerShell 命令差异修正总结

## 1. 背景

T050.3a 创建了命令权限白名单策略，其中 A 类低风险自动执行命令包含了 `Test-Path`、`Get-Content`、`Select-String` 等 PowerShell 命令。

在 `/permissions` 配置建议中，这些 PowerShell 命令被错误地放入了 `Bash(...)` 白名单。

## 2. 问题现象

T051 自动验收中，Claude Code 在 Bash 环境执行了 `Test-Path`，导致：

```text
/usr/bin/bash: line 1: Test-Path: command not found
```

类似问题也出现在 `Get-Content` 和 `Select-String`。

## 3. 原因分析

- `Test-Path`、`Get-Content`、`Select-String` 是 PowerShell 命令，不能直接在 Bash 中执行。
- Claude Code 在 Windows 项目中可能通过 `Bash(...)` 执行命令。
- 命令权限白名单中没有区分 Bash 和 PowerShell，导致 PowerShell 命令被错误加入 Bash 白名单。

## 4. 修正内容

- 修正 `/permissions` 配置建议：`Bash(Test-Path*)` → `Bash(test -f*)`，`Bash(Get-Content*)` → `Bash(tail -n*)`，`Bash(Select-String*)` → `Bash(grep -n*)`
- A 类命令区分为 Bash 文件检查和 PowerShell 文件检查两节
- 追加第 14 节 Bash 与 PowerShell 命令差异对照
- 更新 full task loop 协议的 Shell Compatibility 说明
- 更新经验和避坑记录

## 5. Bash 推荐命令

| 用途 | 命令 |
|------|------|
| 文件存在检查 | `test -f <path> && echo "OK"` |
| 文件尾部读取 | `tail -n <number> <path>` |
| 文件头部读取 | `head -n <number> <path>` |
| 文本搜索 | `grep -n "<pattern>" <path>` |
| 路径分隔符 | 正斜杠 `/` |

## 6. PowerShell 推荐命令

| 用途 | 命令 |
|------|------|
| 文件存在检查 | `Test-Path <path>` |
| 文件尾部读取 | `Get-Content <path> -Encoding UTF8 -Tail <number>` |
| 文件头部读取 | `Get-Content <path> -Encoding UTF8 -Head <number>` |
| 文本搜索 | `Select-String -Path <path> -Pattern "<pattern>"` |
| 路径分隔符 | 反斜杠 `\` |

## 7. 后续提示词规范

后续自动化提示词优先使用 Bash 版本验收命令，除非明确指定 PowerShell。

自动执行授权段落应使用：

- `test -f <path>` 代替 `Test-Path <path>`
- `tail -n <number> <path>` 代替 `Get-Content <path> -Tail <number>`
- `grep -n "<pattern>" <path>` 代替 `Select-String -Path <path> -Pattern "<pattern>"`
