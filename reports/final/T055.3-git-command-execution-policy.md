# T055.3 Git 备份命令执行规则总结

## 1. 背景

在执行 T055.2 Git 备份任务时，Claude Code 曾尝试执行：

```bash
cd "E:/github_project/multi-agent-runner" && git add docs/tasks.md reports/dev/T055.1-dev-report.md && git commit -m "docs: add T055.1 dev report" && git push
```

该命令触发了 Claude Code 安全确认：

```
Compound commands with cd and git require approval to prevent bare repository attacks
```

这说明当前 allowlist 中虽然有单独的 `cd` 和单独的 Git 命令规则，但 `cd && git` 复合命令不在白名单范围内，也不应该被加入白名单。

## 2. 问题现象

- 单独 `cd` 可以作为低风险命令自动执行
- 单独 `git add` / `git commit` / `git push` 可以在 Git 备份任务中自动执行
- 但 `cd && git add && git commit && git push` 复合命令会触发安全确认
- 复合命令不应被加入 allowlist

## 3. 风险分析

`cd && git` 复合命令的风险：

1. 可能绕过路径安全检查
2. 可能在错误目录执行 Git 写入操作
3. 可能导致 bare repository 问题
4. 一条命令中包含多个写入操作，难以定位失败点
5. 不符合逐条执行、逐条确认的安全原则

## 4. 允许的执行方式

在明确 Git 备份任务中，按顺序逐条执行：

```bash
pwd                                    # 确认当前目录
git status --short                     # 检查待提交内容
git check-ignore -v .env               # 确认 .env 不被提交
git diff --stat                        # 查看变更统计
python runner.py                       # 确认下一个 pending
git add <explicit-files>               # 暂存指定文件
git commit -m "<message>"              # 提交
git push                               # 推送
git status --short                     # 确认工作区 clean
git log --oneline -1                   # 确认提交成功
```

每条命令独立执行，便于定位失败点和控制风险。

## 5. 禁止的复合命令

```bash
# 禁止：cd + git 写入组合
cd "E:/github_project/multi-agent-runner" && git add .
cd "E:/github_project/multi-agent-runner" && git commit -m "..."
cd "E:/github_project/multi-agent-runner" && git push

# 禁止：多个 Git 写入命令合并一行
cd <project-path> && git add . && git commit -m "..." && git push
git add . && git commit -m "..." && git push
```

不应把此类复合命令加入全局 allowlist。

## 6. 目录不正确时的处理

执行 Git 备份任务前必须先运行 `pwd`。

如果当前目录不是 `E:/github_project/multi-agent-runner`：

- 必须停止并报告
- 不得使用 `cd && git` 复合命令继续执行
- 应由用户手动切换目录后再执行 Git 备份任务

## 7. 后续提示词规范

后续 Git 备份任务提示词应：

1. 明确要求逐条执行 Git 命令
2. 明确禁止 `cd && git` 复合命令
3. 先要求执行 `pwd` 确认目录
4. 将 `git add`、`git commit`、`git push` 列为独立的允许命令
