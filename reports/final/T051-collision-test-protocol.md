# T051 碰撞检测测试协议报告

## 1. 背景

T047 设计了重力下落测试协议，用于验证 G006 简单重力下落逻辑。G006 已完成完整闭环（Developer / Tester / Reviewer / Main Agent）。

G007 将引入玩家与平台基础碰撞逻辑，使玩家落到平台上时停止下落。碰撞是游戏核心交互基础，后续平台滚动、游戏失败条件、关卡设计都依赖碰撞系统。

## 2. 当前问题

T047 的重力测试协议可以验证：

- 重力常量和垂直速度
- 玩家 y 坐标随时间增加
- 重力更新集成到 gameLoop 中

但当前 Tester 还不能验证：

- 玩家是否会在平台上停止下落
- 是否存在碰撞检测函数
- 碰撞逻辑是否正确判断玩家底部位置和水平范围
- 落到平台后 y 坐标和垂直速度是否修正
- 玩家是否会穿透平台

G007 引入碰撞后，需要新增碰撞检测测试协议。

## 3. 协议目标

定义 Tester Agent 如何检查 G007 玩家与平台基础碰撞逻辑：

- 碰撞检测函数是否存在
- 碰撞逻辑是否在下落时执行
- 平台数据是否正确使用
- 玩家底部位置判断
- 水平范围重叠判断
- 从上方落到平台的判断
- 落到平台后 y 坐标修正
- 落到平台后垂直速度归零
- 防穿透检查
- 不越界实现平台滚动、随机平台和失败条件

## 4. 测试范围

### 4.1 G007 测试范围

- 玩家下落时能检测到平台
- 玩家落到平台上时停止下落
- 玩家不会穿过平台
- 玩家站在平台上时垂直速度归零
- 碰撞逻辑集成到 gameLoop 中
- 平台数据来源于 G005 的固定平台布局

### 4.2 不测试什么

- 平台滚动
- 随机平台
- 游戏失败条件
- 游戏胜利条件
- 角色技能系统
- 复杂物理系统
- 微信小游戏 / 抖音小游戏适配

## 5. 检查项设计

共 6 组 18 个检查项：

| 组别 | 名称 | 检查项数量 | 必需项数量 |
|------|------|------------|------------|
| C 组 | 碰撞函数 / 状态检查 | 3 | 2 |
| P 组 | 平台数据检查 | 3 | 3 |
| L 组 | 玩家落到平台检查 | 3 | 3 |
| S 组 | 停止下落检查 | 3 | 3 |
| T 组 | 防穿透检查 | 3 | 3 |
| O 组 | 范围限制检查 | 3 | 3 |

### C 组：碰撞函数 / 状态检查

| 编号 | 测试项 | 必需 |
|------|--------|------|
| C-01 | 碰撞检测函数存在 | 是 |
| C-02 | 碰撞逻辑在下落时执行 | 是 |
| C-03 | 存在站立状态或平台接触状态 | 否 |

### P 组：平台数据检查

| 编号 | 测试项 | 必需 |
|------|--------|------|
| P-01 | 平台数据存在 | 是 |
| P-02 | 平台有 x/y/width/height 信息 | 是 |
| P-03 | 碰撞逻辑遍历平台 | 是 |

### L 组：玩家落到平台检查

| 编号 | 测试项 | 必需 |
|------|--------|------|
| L-01 | 判断玩家底部位置 | 是 |
| L-02 | 判断玩家水平范围重叠 | 是 |
| L-03 | 判断从上方落到平台 | 是 |

### S 组：停止下落检查

| 编号 | 测试项 | 必需 |
|------|--------|------|
| S-01 | 落到平台后修正 y 坐标 | 是 |
| S-02 | 落到平台后垂直速度归零 | 是 |
| S-03 | 更新玩家 DOM 位置 | 是 |

### T 组：防穿透检查

| 编号 | 测试项 | 必需 |
|------|--------|------|
| T-01 | 使用上一帧位置或下落方向避免误判 | 是 |
| T-02 | 玩家不能直接穿过平台 | 是 |
| T-03 | 碰撞只在下落时触发 | 是 |

### O 组：范围限制检查

| 编号 | 测试项 | 必需 |
|------|--------|------|
| O-01 | 不包含平台滚动逻辑 | 是 |
| O-02 | 不包含随机平台生成 | 是 |
| O-03 | 不包含游戏失败条件 | 是 |

## 6. PASS / FAIL / BLOCKED 规则

| 条件 | 结果 |
|------|------|
| 全部必需项通过 | PASS |
| 任一必需项失败 | FAIL |
| script.js 不存在或无法读取 | BLOCKED |
| 平台数据不存在 | BLOCKED |
| 范围越界实现明显存在 | FAIL |

## 7. 完成证据路径

碰撞行为检查报告保存路径：

```text
<project-root>/reports/test/<task-id>-collision-test-report.md
```

例如：

```text
projects/down-100-floors-game/reports/test/G007-collision-test-report.md
```

Main Agent 应同时读取基础测试报告、碰撞测试报告、开发报告、审查报告和综合决策报告。
