# T050.3a Claude Code 安全命令自动执行白名单总结

## 1. 背景

T050.3a 之前，项目协议中已设计了命令权限策略的概念（A/B/C/D 四类），但没有落地为具体文档和配置建议。

Claude Code 在执行低风险命令时经常要求人工确认，影响了自动化落地效率。

## 2. 当前问题

- 低风险只读命令（`git status`、`python runner.py`）每次都需要人工确认
- 缺少明确的权限白名单文档
- 缺少 `/permissions` 配置建议
- 不清楚哪些命令可以安全自动化

## 3. 权限分级

| 级别 | 说明 | 默认行为 |
|------|------|----------|
| A 类 | 低风险自动执行 | 自动执行，无需确认 |
| B 类 | 仅 Git 备份任务中自动执行 | 只在明确 Git 备份任务中自动执行 |
| C 类 | 需要人工确认或任务显式授权 | 默认需确认，任务明确授权时可执行 |
| D 类 | 禁止自动执行 | 任何情况下都禁止自动执行 |

## 4. A 类自动执行命令

- Git 只读：`git status`、`git diff --stat`、`git log --oneline`、`git branch --show-current`、`git check-ignore`
- Python 检查：`python runner.py`、`python -m py_compile`
- PowerShell 文件检查：`Test-Path`、`Get-Content`、`Select-String`
- 限制：不得打印 `.env` 内容和 API Key

## 5. B 类 Git 备份命令

- `git add .`、`git commit -m`、`git push`
- 只在明确 Git 备份任务中允许
- 执行前必须检查 `git status --short` 和 `git check-ignore -v .env`
- 执行后必须确认工作区 clean

## 6. C 类需确认命令

- `git restore`、`Remove-Item`、`Move-Item`、`Copy-Item`
- 默认需要人工确认
- 只有任务提示词明确授权时才可执行

## 7. D 类禁止命令

- `git reset --hard`、`git clean -fd`、`rm -rf`、`Remove-Item -Recurse`
- 禁止打印 `.env` 内容和 API Key
- 禁止删除项目目录
- 禁止提交 secrets

## 8. /permissions 配置建议

优先允许 A 类模式：

```
Bash(git status*)
Bash(git diff --stat*)
Bash(git log --oneline*)
Bash(python runner.py*)
Bash(python -m py_compile*)
```

Git 备份命令建议只在备份任务中临时允许。

## 9. 安全边界

- `.env` 和 API Key 永远不能提交或打印
- 删除命令不在白名单中
- Git 备份权限不得扩展到普通开发任务
- full loop 遇到 API 失败、BLOCKED 或 D 类命令需求时必须停止
