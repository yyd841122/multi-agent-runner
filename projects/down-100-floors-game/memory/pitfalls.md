# Pitfalls

## G006 简单重力下落注意事项

1. G006 只代表简单重力下落完成，不代表平台碰撞完成。
2. 不要在 G006 阶段加入平台站立逻辑。
3. 不要重复执行已完成的 G006 Developer 阶段。
4. Reviewer BLOCKED 通常是环境或 API 问题，不应直接归因于游戏代码。
5. G006 后续需要 gravity tester 进行专项验证。

## G006 完整闭环注意事项

1. G006 完成不代表平台碰撞完成。
2. G006 完成不代表玩家可以站在平台上。
3. 不要在 G006 后重复执行 Developer 阶段。
4. 后续 G007 应只做平台基础碰撞，不应同时加入滚动和失败条件。
5. 当前 G006 仍缺少 gravity tester 专项测试，应在后续补齐。

## G002 自动执行注意事项

1. **不要在 G002 阶段实现完整游戏逻辑。** G002 只做基础页面布局。
2. **不要提前实现角色技能系统。** 角色系统是后续长期目标。
3. **不要提前做微信小游戏或抖音小游戏适配。** 当前只做 Web MVP。
4. **不要使用真实人物姓名、照片或明显身份特征。** 角色设计应偏纪念和原创。
5. **后续新增玩法时，每一步都必须小而可验证。** 遵循与主框架相同的小步验证原则。

## G003 自动执行注意事项

1. **G003 阶段只显示玩家，不实现键盘移动。** 移动是 G004 或后续任务。
2. **不要提前实现平台、重力、碰撞。** 等基础角色完成后逐步引入。
3. **不要提前实现角色技能系统。** 角色系统是后续长期目标。
4. **不要提前适配微信小游戏或抖音小游戏。** 当前只做 Web MVP。
5. **玩家角色当前只是占位方块，不代表最终角色设计。** 后续可迭代样式。

## G003 DeepSeek Reviewer 审查注意事项

1. **Reviewer 审查通过不代表后续玩法一定正确。** G003 只审查玩家显示，未涉及游戏逻辑。
2. **当前只审查静态文件和任务边界，尚未执行自动测试。** Tester Agent 尚未接入。
3. **后续 G004 / G005 增加移动、平台、重力时，必须加入 Tester Agent。** 静态审查无法覆盖运行时逻辑。
4. **审查报告应保留，不能只看终端输出。** 报告是可追溯证据的一部分。
5. **如果 DeepSeek 输出格式不符合协议，应优先修 prompt 或解析逻辑。** 不要在解析层做过多兼容。

## G003 完整证据链闭环注意事项

1. **G003 COMPLETE 只代表玩家显示任务完成，不代表游戏整体完成。** 游戏还需要移动、平台、重力等功能。
2. **后续 G004 不应同时引入重力、平台、碰撞。** 保持小任务边界。
3. **后续任务必须继续保持 Developer / Tester / Reviewer / Main Agent 证据链完整。** 三方证据缺一不可。
4. **如果 Tester 或 Reviewer 任一失败，不应直接进入下一任务。** 必须修复后重新验证。
5. **综合决策报告必须保留，方便后续追踪。** 报告是可追溯证据的一部分。

## G004 玩家键盘左右移动注意事项

1. **当前只实现左右移动，不代表角色运动系统完整。** 后续还需重力、平台、碰撞等能力。
2. **后续需要验证移动边界是否在真实交互中可靠。** 静态检查无法验证运行时边界行为。
3. **后续实现重力时，要避免和左右移动逻辑冲突。** 重力和水平移动应分别处理。
4. **后续实现平台碰撞时，需要重新检查玩家位置更新逻辑。** 碰撞检测可能影响移动计算。
5. **不要在角色移动阶段引入复杂技能系统。** 角色系统是后续长期目标。

## 第三阶段游戏验证避坑总结

### 注意事项

- G004 完成不代表完整运动系统完成。
- 后续需要增强对键盘移动的真实交互测试。
- 平台、重力、碰撞应拆成独立任务。
- 不要过早加入角色技能系统。
- 不要过早进入小游戏平台适配。

## G005 Developer 特殊完成注意事项

### 避坑记录

1. G005 已完成 Developer 阶段，不应重复执行 `run-project-next`。
2. 后续应从 Tester 阶段继续，而不是重新开发。
3. API 429 属于模型调用限制，不一定代表业务代码失败。
4. 继续前应确认平台数量（5 个）、样式和任务边界。
5. 不应在 G005 阶段引入重力、碰撞、平台滚动或随机生成。
6. 平台当前使用固定布局（PLATFORM_LAYOUT），后续如需随机生成应作为独立任务。

## G005 基础平台显示注意事项

### 避坑记录

1. G005 只代表平台"显示"完成，不代表平台已经可交互。
2. 后续 G006 重力下落时，要避免直接加入碰撞检测。
3. 后续 G007 平台碰撞时，应基于 G005 平台布局继续扩展。
4. 不要把固定平台显示误认为平台滚动或随机生成。
5. 不要重复执行已经完成的 G005 Developer 阶段。

## 第四阶段游戏验证避坑总结

### 注意事项

- G005 完成不代表平台已经可以承载玩家。
- G005 完成不代表已经有碰撞检测。
- 后续重力和碰撞要拆成独立任务。
- 不要重复执行已完成的 G005 Developer 阶段。
- 不要在当前阶段加入随机平台或角色技能系统。
- 不要过早进入微信小游戏 / 抖音小游戏适配。

## G007 玩家与平台基础碰撞注意事项

- G007 完成不代表平台滚动完成。
- G007 完成不代表游戏失败条件完成。
- 后续 G008 或后续任务应继续保持小步开发。
- 碰撞检测应避免侧面或底部误判。
- 不要把平台滚动、随机生成和失败条件混入基础碰撞任务。
